REST API - правила построения роутов: 1. Существительное. (Не глагол). 2. Множественном числе. "/users" 3. Необходимое действие определяет тип запроса: - GET: получить один или все; - POST: добавить; - PUT: обновить полностью; - PATCH: обновить частично; - DELETE: удалить. 4. Запрос к отдельному члену сущности пишется через "/:id": - GET "/users/:id" - получить одного - PUT: "/users/:id" - обновить одного полностью - PATCH: "/users/:id" - обновить одного частично - DELETE: "/users/:id" - удалить одного 5. Разные версии роутом писать через /v_номер_версии/route: /v1/users/ /v2/users/ Какой запрос безопасный? Какой запрос идемпотентный? REST API - постороение ответов: 1. Код ответа определяет как прошло требуемое действие на сервере - по планируемому сценарию или нет 200 - все ок. 201 - Created - создано в случае POST запроса 204 - No content - в случае успешного DELETE 301 - страница перемещена навсегда 302 - страница перемещена временно 400 - Bad Request - нельзя выполнить потому что данные в запросе искажены (их нельзя нормально считать) 401 - Не авторизирован 403 - Нет доступа (авторизирован, но не хватает прав) 404 - ресурс не существует 429 - Слишком много запросов, поэтому ваш отклонен 500 - ошибка сервера (на сервере серьезные проблемы). 2. Что отправлять в ответ: - В случае успешного POST запроса отправляйте назад добавленный в базу объект (с id); - В случае успешного PUT запроса отправляйте назад измененный объект из базы - в случае успешного DELETE запроса отправляйте назад удаленный объект или 204
